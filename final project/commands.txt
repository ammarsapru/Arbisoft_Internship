WAYPOINT - WINDOWS POWERSHELL COMMANDS
=====================================

Run every command from the project root:
C:\Users\ammar\OneDrive\Desktop\personal\arbisoft\final_project_alternative


FIRST-TIME SETUP (run once)
---------------------------

# Create the Python virtual environment and install backend dependencies.
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e ".[dev]"

# Install frontend dependencies.
Set-Location frontend
npm.cmd install
Set-Location ..

# Create the backend-only environment file, then edit .env and add your key.
Copy-Item .env.example .env

# Primary provider for model-backed Ask, Onboarding, mastery, missions, and issue investigation:
# ANTHROPIC_API_KEY=your_key_here
# WAYPOINT_MODEL=claude-sonnet-5

# Claude Code subscription fallback (enabled by default):
# First authenticate once with: claude login
# WAYPOINT_CLAUDE_CODE_FALLBACK=1
# WAYPOINT_CLAUDE_CODE_MODEL=sonnet
# WAYPOINT_CLAUDE_CODE_MAX_TURNS=4
# WAYPOINT_PROVIDER_STARTUP_PROBE=1

# Optional for higher GitHub issue-history API limits:
# GITHUB_TOKEN=your_token_here

# Optional location for persisted chat and analysis-session state:
# WAYPOINT_STATE_PATH=.waypoint-data/waypoint.sqlite3


START THE APPLICATION FOR DEVELOPMENT
-------------------------------------

Open PowerShell terminal 1 in the project root and start the backend:

.\.venv\Scripts\python.exe -m uvicorn backend.app.main:app --host 127.0.0.1 --port 8000 --reload

Open PowerShell terminal 2 in the project root and start the frontend:

Set-Location frontend
npm.cmd run dev

Then open the UI in a browser:
http://127.0.0.1:5173

Backend API documentation:
http://127.0.0.1:8000/docs

The backend reads .env from the project root when it starts. Restart the backend
after changing the key. The key must never be placed in frontend/.env or use VITE_.


OPTIONAL: START THE BACKEND WITH EXTENSIVE DIAGNOSTIC LOGS
----------------------------------------------------------

Run these commands in backend terminal 1 before the Uvicorn command:

$env:ONBOARD_LOG_LEVEL="DEBUG"
$env:ONBOARD_TRACE_FUNCTIONS="1"
$env:ONBOARD_LOG_VALUE_LIMIT="4000"
.\.venv\Scripts\python.exe -m uvicorn backend.app.main:app --host 127.0.0.1 --port 8000 --reload

For the extremely noisy maximum Python call trace, also run this before Uvicorn:

$env:ONBOARD_MAX_TRACE="1"


PRODUCTION-LIKE SINGLE-SERVER MODE (optional)
---------------------------------------------

# Build the frontend, then let FastAPI serve both the UI and API on port 8000.
Set-Location frontend
npm.cmd run build
Set-Location ..
.\.venv\Scripts\python.exe -m uvicorn backend.app.main:app --host 127.0.0.1 --port 8000

Then open:
http://127.0.0.1:8000


STOPPING THE SERVERS
--------------------

Press Ctrl+C once in each terminal running a server.


START THE MCP CONNECTOR (LOCAL STDIO)
------------------------------------

.\.venv\Scripts\python.exe -m backend.app.mcp_server --transport stdio


START THE MCP CONNECTOR (STREAMABLE HTTP)
-----------------------------------------

$env:WAYPOINT_MCP_HOST="127.0.0.1"
$env:WAYPOINT_MCP_PORT="8010"
.\.venv\Scripts\python.exe -m backend.app.mcp_server --transport streamable-http

# The MCP URL is http://127.0.0.1:8010/mcp
# Optional protocol inspector:
npx.cmd -y @modelcontextprotocol/inspector
# Run the local retrieval quality benchmark
.\.venv\Scripts\python.exe benchmarks\run_retrieval.py --repository . --output benchmarks\results\retrieval-latest.json
# Deterministic unit suite (40 tests)
.\.venv\Scripts\python.exe -m pytest -m unit --no-cov

# Deterministic integration suite (33 tests)
.\.venv\Scripts\python.exe -m pytest -m integration --no-cov

# Whole-backend coverage gate followed by per-core-feature gate
.\.venv\Scripts\python.exe -m pytest
.\.venv\Scripts\python.exe -m verification.scripts.core_coverage_report
.\.venv\Scripts\python.exe -m verification.scripts.coverage_report
