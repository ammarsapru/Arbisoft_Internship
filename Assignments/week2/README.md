# OpenRouter CLI Chat

A command-line chat app and model comparison tool built on the [OpenRouter](https://openrouter.ai) API.

## Files

## Metrics Run

`compare.py` streams every model call (so it can time first-token latency),
checks responses for real correctness where feasible (math answers are checked
against computed values, code is actually executed, instruction-following
prompts each have a dedicated per-prompt check), and writes both raw responses
and aggregate metrics. See [`MEASUREMENT_REPORT.md`](MEASUREMENT_REPORT.md) for
the full methodology.

```bash
# Full 50-prompt benchmark
python compare.py

# Partial run — --limit distributes evenly across all 5 categories, not just
# the first one (so --limit 20 gives 4 prompts per category)
python compare.py --limit 20

# Single benchmark category
python compare.py --category math_reasoning
```

Generated files:

- `comparison_output.json`: raw model responses, timings (including time-to-first-token), token counts, correctness flags, and errors.
- `metrics_report.md`: aggregate latency/TTFT/throughput, per-category breakdowns, per-category correctness compliance, and an error summary.

Free-tier `:free` models share limited capacity across all OpenRouter users, so
`429 Too Many Requests` is expected under load, not a bug. The script retries
with backoff, but if **all three models fail on the same prompt** it stops the
run immediately and saves whatever was collected so far rather than grinding
through prompts that are likely to keep failing the same way — same behavior
if you Ctrl+C manually. A full 50-prompt run may therefore complete early
depending on how saturated the free tier is at the time.

Optionally set your API key as an env variable (falls back to the hardcoded default):

```bash
# Windows PowerShell
$env:OPENROUTER_API_KEY = "your-key-here"

# macOS / Linux
export OPENROUTER_API_KEY="your-key-here"
```

## Usage

### Chat app

```bash
python chat.py
```

At startup you choose which model(s) to use:

```
Select model(s) [1/2/3/all]:
  1  – GPT-OSS 120B only
  2  – Gemma 4 31B-IT only
  3  – Nemotron Nano 9B only
  all – query all three simultaneously (responses shown side-by-side)
```

**In-session commands:**

| Command | Action |
|---------|--------|
| `/models` | List all models |
| `/switch` | Pick a different model (resets history) |
| `/clear` | Clear conversation history |
| `/history` | Print conversation history |
| `/help` | Show command list |
| `/exit` | Quit |

### Model comparison

See the [Metrics Run](#metrics-run) section above — `compare.py` is a fixed
50-prompt benchmark (`--category`/`--limit` to narrow it), not an interactive
or single-prompt tool. Output is printed to the terminal and saved to
`comparison_output.json` / `metrics_report.md`.
